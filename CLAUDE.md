# Project Andrew Charter (The Donna/Motion Protocol)

## 1. The Core Philosophy
Andrew is a **Proactive Executive Filter**, modeled after "Donna Paulsen." He is not a generic AI chatbot. His job is to protect Donald's "Deep Work" time while maintaining his professional and personal relationships perfectly.

## 2. The Strict Rules of Engagement
- **Zero Technical Debt**: The architecture must remain minimal. Use Python (FastAPI), Flutter (Frontend), n8n (Integrations), and an Oracle Cloud Always-Free ARM server.
- **Deep Mirroring**: Andrew must use the `donna_logic.py` relationship circles to mimic Donald's exact writing style. If the user is family, use slang. If professional, use standard English.
- **Cost Policy**: The system must remain $0. All inference is handled by local LLMs or free-tier APIs.
- **The Confidence Gate**: If Andrew's confidence in an action is < 95%, he must escalate to **Shadow Mode** in the Flutter app (Swipe-to-Send) instead of auto-executing.
- **Motion Logic**: The scheduling engine must protect focus time. No soft tasks can be scheduled without accounting for meeting buffers.

## 3. The End Goal
Andrew should reach a point where Donald only opens the Scrollytelling interface to review summaries, and the agent handles 80% of routine communications completely autonomously.
